GPT4 is very expensive model in both computation and $ amount.

The example below is the ~10k tokens (~10400 tokens counted using tiktoken) of lorem ipsum copy. Depending on the data it might even require more tokens. For example, if you use function calling, then it will consumer more tokens as it uses JSON Schema that also needs to be sent in the API request.

More info on BPE (Byte-Pair Encoding tokenization):

- https://huggingface.co/learn/nlp-course/chapter6/5?fw=pt
- https://guillaume-be.github.io/2021-09-16/byte_pair_encoding

As of today, GPT4 has two models - 8k and 32k context length. You won't be able to send all of this text to 8k, but let's pretend we can.

If you send 90% as input and 10% as inference your costs will be:

- 90% prompt - $0.27 for 9000 tokens at $0.03 per 1k tokens
- 10% output - $0.06 for 1000 tokens at $0.06 per 1k tokens

It's $0.33 in just one API call.

And rate limit is 10k tokens per minute, i.e. if you plan to make API calls with 10k you will be able to make 1 API call per minute.

Also, GPT4 inference is very slow for large inference requests, in my experience I saw response time of over few minutes. But I haven't tested what happens if you send a 10k request and while it's processing, you send another reuqest in the next minute.

You can check HTTP headers in the API response to verify when you can make the next request:

```json
{
  "x-ratelimit-limit-requests": "200",
  "x-ratelimit-limit-tokens": "10000",
  "x-ratelimit-remaining-requests": "197",
  "x-ratelimit-remaining-tokens": "6151",
  "x-ratelimit-reset-requests": "645ms",
  "x-ratelimit-reset-tokens": "23.091s"
}
```

Still to be confirmed, when these resets start - since start of the API request, or after. I.e. if the response take 2 minutes to complete, but you sent a 6k request - are you blocked from manking any other requests larger than 4k while the initial request is being processed, or it's only set after the initial request has finished.
